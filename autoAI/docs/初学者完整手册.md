# autoAI 初学者完整手册

这是一份给“第一次接触软件项目”的读者准备的文档。  
目标很简单：你看完后，能自己完成 3 件事：

1. 正常启动并使用项目  
2. 知道它为什么能工作（原理）  
3. 知道后续应该怎么继续开发（路线）  

---

## 1) 项目是做什么的（先建立直觉）

`autoAI` 不是去调用官方 API Key，而是用**网页自动化**去操作你已登录的 AI 网页账号。  
你可以把它理解成：

- 你有多个 AI 网站账号（ChatGPT、Claude、DeepSeek…）
- `autoAI` 帮你自动打开这些网页、自动输入、自动点击发送、自动抓取回复
- 最终你在一个统一界面里使用多个 AI

如果你是做工作流平台的人，你还可以通过本地 Adapter 把它当成本地 API 服务使用。

---

## 2) 零基础操作：从 0 到可用

### 第一步：安装依赖

在项目根目录执行：

```bash
npm install
```

### 第二步：启动桌面应用

```bash
npm run dev
```

如果你要使用 Playwright 自动化路径（更接近生产策略）：

```bash
npm run dev:playwright
```

### 第三步：在应用里添加 AI 站点

1. 打开资源页（Resources）  
2. 选择 ChatGPT / Claude / DeepSeek 等卡片  
3. 在弹出的网页登录页完成登录  
4. 返回聊天页开始发送消息

## 3) 如果你要“像 API 一样调用”

项目已包含本地 Adapter（OpenAI 兼容最小接口）。

默认地址：

- `http://127.0.0.1:8787`

常用接口：

- `GET /health`
- `GET /v1/models`
- `POST /v1/chat/completions`

示例（非流式）：

```bash
curl http://127.0.0.1:8787/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-v4-pro",
    "messages": [
      {"role":"system","content":"You are a helpful assistant."},
      {"role":"user","content":"你好，请做个自我介绍"}
    ],
    "stream": false
  }'
```

如果你需要完整字段说明，查看：`docs/本地Adapter接口说明.md`

## 4) 背后原理（不讲代码术语版本）

一次“发送问题 -> 得到回答”大致会经过 4 层：

1. **输入层**：把你的问题注入到网页输入框并触发发送  
2. **提取层**：优先走网络流拦截，失败再走 DOM 观察兜底  
3. **诊断层**：失败会被结构化记录（不是只留一句自然语言日志）  
4. **修复层**：可重试问题会触发有限自动修复（有护栏）

你可以记住一句话：  
**能提取、能定位、能回归，这个项目才能长期稳定。**

## 5) 日常运维：你每天只要跑一个命令

```bash
npm run regression:loop
```

它会做：

1. 类型检查  
2. 测试  
3. 导出失败快照  
4. 按门禁规则判定 pass/fail

关键输出文件：

- `artifacts/regression-loop.log`
- `artifacts/failure-snapshot.json`
- `artifacts/failure-snapshot.events.ndjson`

---

## 6) 出问题时怎么排查（按顺序）

### A. 先判断是“系统问题”还是“站点问题”

看 `failure-snapshot.json`：

- `summary.byErrorCode`：哪类错误最多
- `summary.byStage`：卡在哪个阶段
- `gate.passed`：门禁是否通过

### B. 再看一条具体失败事件

重点字段：

- `errorCode`
- `stage`
- `sendSeq`
- `path`
- `siteId`
- `retryable`

### C. 最后决定动作

- `retryable=true`：优先重试或自动修复策略  
- `retryable=false`：优先人工检查（登录状态、页面变化、网络证书）

## 7) 继续开发：推荐路线（按风险从低到高）

### 路线 1：先“稳”

1. 降低 `UNKNOWN` 比例（补错误码覆盖）  
2. 提高分类准确率（failure-classifier）  
3. 收敛最常见 3 个失败场景

### 路线 2：再“快”

1. 优化自动修复命中率  
2. 增加并发控制与退避策略  
3. 让回归脚本更快反馈（分层测试）

### 路线 3：再“强”

1. 完成 `stream=true`（SSE 流式返回）  
2. 增加任务取消能力  
3. 做工作流首页编排（像 AI Studio 的流程化体验）

## 8) 你可能会问（FAQ）

### Q1：我不会写代码，能用吗？
可以。按“第 2 节 + 第 5 节”执行即可。

### Q2：为什么不直接全用官方 API？
这个项目的目标是“基于网页自动化替代 API 场景”，适合已有网页账号、想统一入口的人。

### Q3：我怎么判断今天是否可上线？
跑 `npm run regression:loop`，看门禁是否通过，`gate.passed` 是否为 `true`。

### Q4：要给同事排查问题，最少发什么？
发这 3 个文件就够：

- `artifacts/failure-snapshot.json`
- `artifacts/failure-snapshot.events.ndjson`
- `artifacts/regression-loop.log`

---

## 9) 下一步建议（给你这个项目 owner）

如果你只做一件事，我建议是：  
**优先把 stream=true 补齐，并把错误码覆盖率做成周报。**

这样你既能提升使用体验，也能长期看到稳定性是否在变好。
