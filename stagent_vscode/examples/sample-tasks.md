# 示例任务描述

以下为用户需求示例，可用于测试 Stagent 工作流生成；**非项目文档**。

## 语音聊天室（React Native）

生成一个带语音功能的聊天室 app，能在苹果和安卓手机使用。

**业务目标**  
构建一个面向移动端的多人实时聊天室应用，支持 iOS 与 Android 双平台，以文字消息与语音消息为交流核心，明确排除视频通话或视频消息。

**技术与交付假设（可改）**  
- 交付形态：跨平台移动应用项目，代码组织为 **React Native** 项目。
- 实时消息通道：采用 **WebSocket** 长连接为主。
- 后端：默认 **TypeScript + Express + PostgreSQL**。

**工作流产出要求**  
1. 全局架构决策（技术选型、WebSocket 协议、语音录制/上传流程）。
2. 垂直切片：骨架 → 身份与房间 → 文字消息 → 语音闭环 → 异常处理 → 打磨。

## 微信小程序 + Web 管理后台（二维码绑定视频）

请为我规划并实现一个**微信小程序 + Web 管理后台**的完整项目：

1. **二维码与视频绑定**：生成唯一二维码；拍摄者绑定约 20 秒内视频；其他用户扫码播放。
2. **管理后台**：登录权限、批量/单个生成与下载二维码、统计看板（绑定数、观看量等）。
3. **技术假设**：小程序原生 + Web 后台 + 微信云开发。

## 待办列表 Web（React + Vite + TypeScript）

**任务标题**：做一个极简「待办列表」网页应用（本地可构建）

**功能范围（MVP）**  
1. 首页展示待办列表，支持新增、勾选完成、删除。  
2. 数据仅存浏览器 `localStorage`。  

**工程与验收要求**  
- 可 `npm install`、`npm run build`、`npm test`（占位亦可）。  
- 技术栈：**React + Vite + TypeScript**。

**非目标**：不要小程序、不要后端 API、不要数据库。
